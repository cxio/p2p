# 工作量认证

节点之间的连接需要先执行一个工作量运算，以防止某些攻击，维护网络的良好状态。


## 基本条件

执行工作量哈希运算的因子包含：

- 客户端固定种子（`AppSeed`）值，32字节随机数。可随App升级更新。
- 服务端地址（`ServAddr`）IP:Port 序列化值，让哈希因子依目标不同而不同。
- 实时时间戳。容许 `±5分钟` 的误差，使得客户端&服务器各自计算时相同。
- 一个秘密随机数（`nonce`），由服务端猜测，主要用来回避直接特征验证（流量分析）。
- ClientHello 消息包随机数（`CH.random`），工作量主体因子。
- 临时随机数（`tmpRandom`），仅在需要拥塞控制时才使用，属于二级认证时。

> **注：**
> 工作量哈希算法采用抗ASIC的 `RandomX` 或 `Argon2id`。
> 难度限定（哈希结果前置0位）会削减 CH.random 信息熵，但基本可以忽略。

工作量*难度值*与 `AppSeed` 一样，会内置于客户端（硬编码，仅升级时可变）。


## 响应分级

发起连接一端初始的 ClientHello 中即包含工作量证明，使得首包即认证。服务端验证通过后，即可进行下一步的必要回应。回应分四种情况：

0. 未提供工作量证明，视为非法连接，简单不予理会。
1. 上线协助：对端提供了*初始工作量*证明，但ECH解密失败或无加密。视为初始上线节点请求协助。
2. 一级认证：同上*初始工作量*验证通过，服务端返回证书，开始正常连接。
3. 二级认证：拥塞控制。如果节点负载较大或连入压力大，新连入的节点会被要求进一步的工作量证明。

> **注：**
> 拥塞控制工作量难度随负载压力增大而提升，动态值。


### 初始工作量

运算的随机种子由对端的公钥哈希提供，难度固定，硬编码内置于客户端。

初始工作量认证在连接之前完成，因此随机数种子需要某种确定性。算法如下：

```go
// 时间戳：±5分钟范围
ts := time.Now().Unix()
Timestamp := ((ts + 300) / 600) * 600

// 常数：强化Bit位熵
BigOdd := 0x517cc1b727220a95

// 一级认证：
// 初始请求即开始，首包即证明。
// 工作量：CH.random
// @nonce 小值（1~32），取随机值后即固定
proof = Hash( AppSeed || ServAddr || Timestamp || CH.random || (BigOdd * nonce) )
```


### 拥塞控制

对于正常的节点，在通过了初始工作量认证后，则已成功建立连接。但如果服务节点负载压力大，需要拒绝部分连入以缓解状况，则可以要求新连入的节点执行二级工作量认证。

此时的难度由服务端自行决定，通常与负载正相关（负载大 => 难度高）。

难度值会设置在 ServerHello 的加密扩展中，客户端在确认连接正常之前，需要先检查这个值（是否有该字段或为零）。

```go
// 二级认证：
// 工作量：tmpRandom
proof = Hash( AppSeed || ServAddr || Timestamp || tmpRandom )
```

此时双方已可正常传递消息（加密连接），工作量因子 `tmpRandom` 需要回传到服务器端接受验证。


## 工作量验证

> **参考：**
> 为了实现 ClientHello 消息包定制，以及 ServerHello 消息包数据注入，
> 需要定制标准 TLS 库实现（可能 fork uTLS 修改，详见：[wTLS](https://github.com/cxio/wtls)）。

### 客户端

- 获取服务端地址（IP:Port），创建一个 nonce 的随机值 n（不变）。
- 创建一个32字节的空间，用安全随机数填充，计算哈希结果。反复尝试直到满足目标难度。
- 将该32字节随机序列设置为 ClientHello.random 字段的值。
- 如果是首次上线，请求对端提供其它节点信息，则明文发送 ClientHello 消息包。
- 如果已经获得节点信息（含ECH公钥），则创建加密 ClientHello 消息包发送。

### 服务器

- 接收到客户端发来的握手消息（ClientHello），提取其中的 random 字段值。
- 创建一个32个成员的整数数组，填充随机不重复的 `1...32` 数值序列。
- 顺序迭代数组成员，取其值为 nonce，根据算法计算哈希值，验证难度是否满足。
- 如果最终没有满足难度，即为非法连接请求，简单丢弃忽略。否则根据情况发送响应（功能服务）。

如果决定执行拥塞控制：

- 向返回的 ServerHello 中设置难度字段（值非零）。
- 等待对端返回消息，然后执行二级认证（无nonce，直接验算）。失败则断开连接。
