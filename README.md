# P2P基础网络（p2p-net）

这是一个P2P节点的自由市场，任何应用都可以进入这个市场。

如果一个应用实现了*搜寻应答*接口，它就可以回应传入的搜寻请求：如果自己就是搜寻的目标（*名识*匹配），即回应自己的连系信息，否则转播该请求。这样，就形成了一个被动式的通用搜寻机制，实现了任意P2P应用的节点发现功能。

应用有自己专属的子网，当它们找到了自己的同类节点，即可连入本应用的子网。

成功连入自己应用子网的节点并不需要离开这个市场（资源占用极少），它们是市场的一部分，协助市场传播其它节点的搜寻请求。

如果这个市场成长到足够庞大（百万级连接），那么*暴力发现*就是可行的。这可以让初始节点中心化的机制不再必要，从而实现真正的完全去中心化！


## 功能

实现P2P网络的基本功能和节点搜寻功能，包括：

- 连接维护：测试分享的节点可达性，建立连接以及心跳维护。
- 分享节点：节点间交换彼此缓存的节点信息，更新本地缓存池。支持*毒IP*混入（即大平台节点，用于抗审查）。
- 节点评估：对节点进行评分，不良节点加入黑名单（限时阻止）。
- 节点搜寻：接受任意应用节点对其同类节点的搜寻：转播搜寻请求，直到抵达目标节点（目标节点根据传递的*名识*做判断）。
- 搜寻回应：目标节点回应自己的节点信息，回应消息原路回传。这是一种逐级递减逻辑，避免洪泛。

> **注：**
> 在实际的实现中，节点搜寻功能由通用的**广播查找**接口实现。

设计详见 [docs/design](docs/design.md)。


### 关于毒IP

如果审查者对网络节点进行收集屏蔽，在无法区分IP实际指向的情况下，指向大平台的毒IP也会同时被屏蔽。这种附带伤害可以一定程度抑制审查的力度，从而保护本网络的可用性。

毒IP是混合在分享的节点清单里的，它们对本网络不可用，但我们需要适当接收它们。因此在对分享的节点进行可达性测试时，无需全部测试，简单地取一部分（比如 1/2 ~ 2/3）即可。

> **提示：**
> 在可达性测试时，如果节点不可达也会正常丢弃。局部模糊性策略已经足够。

执行可达性测试的比例应当是可配置的，以适应不同的网络环境和需求。


### 暴力发现

在不追求及时性的情况下，遍历全网IPv4地址来发现节点是可能的。如果需要，用户用一台小设备挂机……

作为一种优化，历史IP可能是有用的：如果节点只是切换到了它附近的位置，以历史IP为原点的小范围搜寻会很高效。

一旦找到某个节点，连入网络就可以获取更多节点了。
